# PRD: 나의 가족 이야기

## 1. 제품 개요

`나의 가족 이야기`는 로그인한 사용자만 가족 사진을 등록하고 감상할 수 있는 개인용 사진 갤러리 웹 애플리케이션이다. 사진은 masonry 레이아웃으로 배치되며, 사용자는 사진을 클릭해 확대 이미지와 설명을 함께 확인할 수 있다.

## 2. 목표

- 가족 사진과 추억을 로그인 사용자에게만 안전하게 보여준다.
- 사진 등록 시 설명을 함께 저장할 수 있게 한다.
- 갤러리를 masonry 레이아웃으로 구성해 다양한 비율의 사진을 자연스럽게 보여준다.
- 사진 클릭 시 확대 보기와 상세 설명을 제공한다.
- 비로그인 사용자는 갤러리 대신 로그인 안내 랜딩 페이지를 본다.

## 3. 대상 사용자

- 가족 사진을 한곳에 모아 보고 싶은 개인 사용자
- 가족 구성원과 추억 사진을 공유하고 싶은 사용자
- 간단한 설명과 함께 사진 기록을 남기고 싶은 사용자

## 4. 핵심 사용자 흐름

1. 사용자가 사이트에 접속한다.
2. 로그인하지 않은 경우 로그인 안내 랜딩 페이지가 표시된다.
3. 사용자가 Clerk 인증으로 로그인한다.
4. 로그인 후 사진 갤러리 페이지가 표시된다.
5. 사용자는 사진을 등록하고 설명을 입력한다.
6. 등록된 사진은 masonry 레이아웃에 추가된다.
7. 사용자가 사진을 클릭하면 확대 이미지와 설명이 표시된다.

## 5. 기능 요구사항

### 5.1 인증

- Clerk를 사용해 로그인, 회원가입, 로그아웃을 제공한다.
- 로그인하지 않은 사용자는 사진 목록, 상세 보기, 등록 기능에 접근할 수 없다.
- 로그인하지 않은 사용자는 로그인 안내 메시지와 로그인 버튼만 볼 수 있다.

### 5.2 사진 갤러리

- 로그인한 사용자는 등록된 사진 목록을 볼 수 있다.
- 사진 목록은 masonry 레이아웃으로 표시한다.
- 사진 카드에는 이미지가 중심이 되며, 필요 시 짧은 제목 또는 설명 일부를 표시할 수 있다.
- 사진이 없는 경우 빈 상태 메시지와 사진 등록 유도 버튼을 보여준다.

### 5.3 사진 등록

- 사용자는 이미지 파일을 업로드할 수 있다.
- 사용자는 사진 설명을 입력할 수 있다.
- 설명은 선택 입력으로 시작하되, 추후 제목, 촬영일, 태그 확장을 고려한다.
- 업로드 실패, 파일 형식 오류, 네트워크 오류에 대한 안내 메시지를 제공한다.

### 5.4 사진 상세 보기

- 사용자가 갤러리의 사진을 클릭하면 확대 보기 모달 또는 상세 페이지가 열린다.
- 상세 보기에는 확대된 사진과 설명이 표시된다.
- 사용자는 닫기 버튼 또는 배경 클릭으로 상세 보기를 종료할 수 있다.

## 6. 비기능 요구사항

- 반응형 UI를 제공해 모바일과 데스크톱에서 모두 사용할 수 있어야 한다.
- 이미지 로딩 중에는 레이아웃 이동을 최소화하기 위해 고정 비율 또는 placeholder를 사용한다.
- 인증이 필요한 데이터는 서버 측에서 사용자 인증 상태를 확인한 뒤 제공한다.
- `.env`에 저장된 Clerk 키와 기타 비밀 값은 저장소에 커밋하지 않는다.

## 7. MVP 범위

- Clerk 로그인 및 로그아웃
- 비로그인 랜딩 페이지
- 로그인 후 masonry 사진 갤러리
- 사진 업로드 및 설명 저장
- 사진 클릭 시 확대 보기와 설명 표시

## 8. 제외 범위

- 가족 구성원별 권한 관리
- 댓글, 좋아요, 공유 링크
- 사진 편집 기능
- 앨범 분류 및 태그 검색
- 관리자 페이지

## 9. 성공 기준

- 비로그인 사용자는 사진 데이터를 볼 수 없다.
- 로그인 사용자는 사진을 등록하고 갤러리에서 확인할 수 있다.
- 사진 클릭 시 확대 이미지와 설명이 정상 표시된다.
- 데스크톱과 모바일에서 갤러리 레이아웃이 깨지지 않는다.

## 10. 향후 확장 아이디어

- 앨범별 사진 분류
- 촬영일, 장소, 인물 태그 추가
- 가족 구성원 초대 기능
- 사진 검색 및 필터
- 클라우드 스토리지 연동
